# Internationalization (i18n)

The app uses **vue-i18n** for client-side translations. Currently supported:

- **en** – English (default)
- **ar** – Arabic (RTL)

## Usage in components

```vue
<script setup lang="ts">
import { useI18n } from 'vue-i18n';

const { t, locale } = useI18n();
</script>

<template>
  <h1>{{ t('products.title') }}</h1>
  <p>{{ t('footer.copyright', { year: 2025 }) }}</p>
</template>
```

Or use the `useLocale()` composable for locale switching and RTL:

```ts
import { useLocale } from '@/composables/useLocale';

const { locale, setLocale, t, isRtl, supportedLocales } = useLocale();
```

## Adding a new locale (e.g. French)

1. **Create the locale file**  
   Copy `resources/js/i18n/locales/en.ts` to `resources/js/i18n/locales/fr.ts` and translate the values.

2. **Register the locale**  
   In `resources/js/i18n/index.ts`:
   - Add `'fr'` to `SUPPORTED_LOCALES`.
   - Import the new messages and add them to `createI18n({ messages: { en, ar, fr } })`.

3. **Add to the locale switcher**  
   The header switcher uses `supportedLocales`, so the new locale will appear automatically.

4. **RTL (optional)**  
   For RTL languages, add the locale code to the `RTL_LOCALES` array in `resources/js/i18n/index.ts`.

## Persistence

The selected locale is stored in `localStorage` under the key `pcommerce_locale` and is applied on load. The `<html>` element gets `lang` and `dir` (e.g. `dir="rtl"` for Arabic) so layout and RTL styles apply correctly.

## Message keys

Message keys are grouped by area: `common`, `nav`, `home`, `products`, `categories`, `vendors`, `cart`, `wishlist`, `checkout`, `footer`, `locale`. Add new keys in each locale file to keep them in sync.
